2 Permanenter Befehl
Befehl Beschreibung
/zeltcmds Zeigt Informationen und Hilfe
/zeltcmds add <Befehl> <Kategorie> <Typ/Parameter> <Defaultpermission> [Nachricht] Fügt einen neuen Befehl hinzu
/zeltcmds addalias <Alias> <Befehl> [Parameter]* Fügt einen neuen Alias hinzu
/zeltcmds examples Fügt einige vordefinierte Befehle hinzu. Überschreibt keine existierenden Befehle
/zeltcmds list [Seite] Listet definierte Befehle auf
/zeltcmds listalias [Seite] Listet definierte Aliase auf
/zeltcmds reload Lädt Konfiguration und Befehle neu
/zeltcmds remove <Befehl> Entfernt einen Befehl
/zeltcmds removealias <Alias> Entfernt einen Befehl

2.1 Information
'/zeltcmds add', '/zeltcmds addalias', '/zeltcmds remove' und '/zeltcmds removealias' ändern sofort die config.yml, womit '/zeltcmds reload' keine Änderungen rückgängig machen kann!

2.2 Permission
zeltcmds.manage

2.3 Defaultpermission
OP

5 Aliase

5.1 Parameter (Sortiert nach Bearbeitungsreihenfolge)

  • <player_name> - Name of executing player (player only)
  • <player_nick> - Displayname of executing player (player only)
  • <player_x> - x-coordinate of executing player (player only)
  • <player_y> - y-coordinate of executing player (player only)
  • <player_z> - z-coordinate of executing player (player only)
  • <player_world> - worldname of executing player (player only)
  • <item_id> - ID of item in hand of executing player (player only)
  • <item_data> - Data of item in hand of executing player (player only)
  • <item_name> - Name of item in hand of executing player (player only)
  • <paramX> - Der Xte übergebene Parameter
  • <param> - Ein Pflichtparameter der den ersten nicht zugewiesenen Parameter verwendet
  • <param?> - Ein optionaler Parameter der den ersten nicht zugewiesenen Parameter verwendet
  • <param*> - Alle übergebenen Parameter die noch keiner Position zugewiesen bekommen haben
  • <cmd> - Trennt zwei Befehle von einander. Leerzeichen sind vor und nach dem Parameter erlaubt, aber nicht nötig
  • <waitX> - Warte X Ticks vor der Ausführung des Befehls. Muss vor dem Befehl stehen

5.2 Beispiele

Beispiel 1: (Einfacher Alias für Befehl)
/zeltcmds addalias zc zeltcmds
> zc ist Alias für zeltcmdsm, womit man /zc [...] anstelle von /zeltcmds [...] verwenden kann

Beispiel 2: (Erweiterter Alias mit konstanten Parametern)
/zeltcmds add m2c port m2c true Woosh!
> Befehl wird für Beispiel verwendet
/zeltcmds addalias zero m2c 0 0
> zero ist ein Teleport für den Spieler zu Position x=0;z=0 ohne weitere Parameter

Beispiel 3: (Erweiterter Alias mit konstanten Parametern und einem dynamischen)
/zeltcmds add m2wc port m2wc true Woosh!
> Befehl wird für Beispiel verwendet
/zeltcmds addalias zerow m2wc <param> 0 0
> zerow ist ein Teleport für den Spieler zu Position x=0;z=0 auf einer Welt die angegeben werden muss
/zerow test
> Teleportiert den Spieler zur Welt 'test' an Position x=0;z=0

Beispiel 4: (Ein Alias aus zwei Befehlen mit einem dynamischen Parameter)
/zeltcmds add toworld port m2w true
/zeltcmds add home port m2h true Home sweet home!
> Befehle werden für Beispiel verwendet
/zeltcmds addalias worldhome toworld <param> <cmd> home
> worldhome teleportiert den Spieler zum Spawn einer bestimmten Welt und von da zu seinem Bedspawn
/worldhome Pangea
> Teleportiert den Spieler zu seinen Bedspawn auf der Welt Pangea

Beispiel 5: (Alias zum erstellen sinnvoller Aliase)
/zeltcmds add tocoord port m2we true
> Befehl wird im Beispiel verwendet
/zeltcmds addalias warpadd zeltcmds addalias <param> tocoord <player_world> <player_x> <player_y> <player_z>
> warpadd erstellt einen Alias mit einem gegebenen Namen, welcher den Spieler zu der Stelle teleportiert, an welcher der Alias erstellt wurde (wie Warps)

Beispiel 6: (Countdown)
/zeltcmds addalias countdown say 3<cmd><wait20>say 2<cmd><wait40>say 1

Beispiel 7: (Items klonen)
/zeltcmds add giveitem give item op
> Befehl wird im Beispiel verwendet
/zeltcmds addalias clone giveitem <param?> <item_id>:<item_data> <param1>
> giveitem klont das Item welches der Spieler in der Hand hält, erstellt die Anzahl welche durch den ersten Parameter gegeben ist und gibt sie optional an den Spieler der als zweiter PArameter übergeben werden kann
